HTML

  • Hypertext Markup Language (HTML) is made up of elements which work together to provide content and the basic framework for all pages on the web.
  • An element is a type of content on a webpage: text elements; image elements; header elements; list elements.
  • Common element types that are used to divide and describe the page: meta-related elements; content division elements; semantic elements.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • !DOCTYPE declaration tells the browser what document type it should exptect, i.e. HTML.
  • html opening tag represents the root, or start, of the HTML document. It can have an attribute inside the tag, e.g. lang="en", which tells the browser the language of the webpage.
  • An HTML attribute defines additional information about an element.
  • meta tags always go inside the head element, and are typically used to specify character set, page description, keywords, author of the document, and viewport settings.
  • The title element defines the title for a webpage. This element is required and is extremely important for SEO.
  • The style element defines the CSS styles associated with a website.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The script element embeds client-side scripts in the HTML document.
  • `charset` is character set, UTF-8 is recommended.
  • `http-equip` element sets an HTTP header for content.
  • The `viewport` element sets the default view for our webpage.
  • Elements that are nested inside other elements are called child elements. The element that holds all the children is known as the parent element.
  • The body element is an example of a semantic HTML element. Semantic HTML elements are elements where the name describes its content and the role it plays on a webpage.
  • It is a good practise to use semantic HTML as it makes it easier to read for other developers and the browser.
  • div to create a division in the page.
  • span to create an inline division.
  • ul = unordered list; ol = ordered list; li = list item.
  • img attribute `src` (location/source, eg url) is most important. `alt` attribute contains a text string that describes the appearance and functionality of the image.
  • The alt text is important for screen readers as well as SEO.
  • Should only have one h1 heading per page.
  • footer element appears at the bottom of the page and usually contains the author, copyright, sitemap, and navigation.
  • section elements are semantic elements that define a section in a document that contains thematic content.
  • HTML uses `id` attribute as a way to identify elements and select them.

CSS

  • Three types of CSS: inline, internal, and external.
  • Inline is within a specific HTML tag and is the least flexible of options.
  • An internal stylesheet can be embedded as as style element in the head element of the HTML file.
  • External stylesheets are the most common method and are connected to the HTML with a link element in the head element.
  • A CSS Rule contains a selector and declarations, where a selector defines the element or attribute and the declarations contain the CSS property with its value.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Two or more elements that will share the same styling can be grouped together.
  • Use wildcard selector, or *, to apply a rule for all elements visible on the page.
  • A period (.) before an element designates it as a class selector whereby any HTML element assigned to that class will gain those property assignments from the CSS.

Git

  • Git commands are different to bash commands. They are used to save and track versions of code.
  • Using git commands we can initiate or create a repo, add our code to the repo, and view the different versions of our code, which are stored in a unique commit.
  • Git can track changes locally. GitHub is cloud based hosting service that allows us to keep track of our projects inside repositories that are stored remotely.
  • Branches are workspaces where you can make changes that don’t affect the live site.
  • Whenever you add, edit, or delete a file, you’re making a commit — adds changes to your branch.
  • We shouldn’t do any coding directly in the main branch, instead we create a feature branch, which is a copy of the main branch.
  • Feature because we are adding a new feature or functionality to our app.
  • In the feature branch we make commits of all the changes we make to the code while working — add a commit message that is descriptive of what changed in the code.
  • Then we push up the changes to our feature branch in our GitHub repository within feature branch.
  • Merge changes into main branch with pull request.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: adds modifications in the current working branch to the staging area.
  • git commit -m "message": saves a new version of the file we are working on. Write imperative command in the present tense.
  • git pull origin main: adds ("pulls") the current main branch from the repo to our local environment.
  • git push origin branch-name: adds ("pushes") our changes in the local environment to the repo.

Javascript

  • A variable is a named container that allows us to store data in our code.
  • String: a sequence of text where the value is enclosed in single quote marks.
  • Number: a number without quotes around them.
  • Boolean: true/false value.
  • Array: a structure that allows you to store multiple values in a single reference.
  • Object: can be anything. Everything in JavaScript is an object and can be stored in a variable.
  • An operator is a mathematical symbol that produces a result based on two values (or variables).
  • Start by declaring a variable with the let keyword.
  • Whereas the style.css file is linked within the head element, the script.js file is linked within the body element.
  • Conditionals are code structures used to test if an expression returns true or not.
  • Instead of repeatedly writing the same code you can define a body of code as a function that executes when you call the function name in the code.
  • JavaScript is a dynamic language with dynamic types. Variables in JavaScript are not directly associated with any particular value type, and any variable can be assigned (and re-assigned) values of all types.
  • JavaScript is also a weakly typed language, which means it allows implicit type conversion when an operation involves mismatched types, instead of throwing type errors.
  • Primitive values are immutable, i.e. cannot be changed — all types except objects.
  • Null values represent the intentional absence of any object value.
  • All primitive types — except null and undefined — have their corresponding object wrapper types. When a property is accessed on a primitive value, JavaScript automatically wraps the value into the corresponding wrapper object and accesses the property on the object instead.
  • Undefined indicates the absence of a value, while null indicates the absence of an object.
  • The BigInt type is a numeric primitive type that can represent integers with arbitrary magnitude.
  • With BigInts you can safely store and operate on large integers event beyond the safe integer limit.
  • Create BigInts by appending n to the end of an integer or by calling the BigInt() function.
  • JavaScript strings are immutable.
  • A symbol is a unique and immutable primitive value and may be used as the key of an Object property. The purpose of symbols is to create unique property keys that are guaranteed not to clash with keys from other code.
  • In computer science, an object is a value in memory which is possibly referenced by an identifier. In JavaScript, objects are the only mutable values. Functions are, in fact, also objects with the additional capability of being callable.
  • There are two types of object properties: the data property and the accessor property. Each property has corresponding attributes.
  • Data properties associate a key with a value.
  • An accessor property associates one of two accessor functions (get and set) to retrieve or store a value.
  • Control flow is the order in which a computer executes code in a script.
  • In JavaScript, code runs from the first statement act the top of the page, down to the last statement at the bottom of the page. However, using conditional statements such as if statements allows us to interrupt control flow.
  • JavaScript arrays are zero indexed, which is to say the fist item has an index of [0], n+1 every other item.
  • Arrays have a special property called length, which is equal to the number of items in the array.
  • At the most basic level, a function is simply a set of instructions that tells the computer how to perform a certain task.
  • A function declaration uses the function keyword and function name to define a function.
  • Need to call the function to execute it.
  • A function’s name is followed by parenthesis.
